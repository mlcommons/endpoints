# uv Migration Design

Migrate from pip/setuptools to uv with lockfile-based supply chain protection.

## Motivation

Current state: all direct dependencies pinned with `==` in `pyproject.toml`. Transitive dependencies are unpinned and unverified. A compromised transitive dep, version hijack, or dependency confusion attack would succeed silently.

`uv.lock` pins and cryptographically hashes every direct and transitive dependency. `uv sync --frozen` refuses to install anything that doesn't match the lockfile.

## Design

### Build Backend

Replace setuptools with `uv_build`:

```toml
[build-system]
requires = ["uv_build>=0.7.6,<0.8"]
build-backend = "uv_build"
```

- Pure Python project, no C extensions -- `uv_build` is sufficient
- `uv_build` auto-discovers `src/` layout -- all `[tool.setuptools]` sections removed
- Package data (`config/templates/*.yaml`) configured via `[tool.uv.build]`
- Exclusion of `livecodebench/_server.py` expressed as a pattern
- pip users: `uv_build` is auto-downloaded from PyPI, transparent

### uv Configuration

```toml
[tool.uv]
default-index = "https://pypi.org/simple"
environments = [
    "sys_platform == 'linux' and platform_machine == 'x86_64'",
    "sys_platform == 'linux' and platform_machine == 'aarch64'",
    "sys_platform == 'darwin' and platform_machine == 'x86_64'",
    "sys_platform == 'darwin' and platform_machine == 'arm64'",
]
```

- Platform restriction to Linux + macOS, x86_64 + arm64 (no Windows -- uvloop has no Windows support)
- Reduces lockfile size by eliminating Windows wheel hashes

### Dependency Structure

Keep `[project.optional-dependencies]` as-is for pip backward compatibility:

- `sql`, `dev`, `test`, `performance` remain as optional extras
- `pip install -e ".[dev,test]"` continues to work
- uv users use `uv sync --extra dev --extra test`

No `[dependency-groups]` migration -- preserves pip compat for all contributors.

### `.python-version`

New file at repo root:

```
3.12
```

uv uses this to auto-select/download the Python interpreter.

### `uv.lock`

Generated by `uv lock`. Contains:

- Every transitive dependency resolved for linux + macOS, x86_64 + arm64
- SHA256 hashes for every wheel and sdist
- Checked into git, reviewed in PRs

### CI: `.github/workflows/test.yml`

All 3 jobs (test, audit, schema-updated) migrate:

- Replace `actions/setup-python` with `astral-sh/setup-uv@v5` (version `0.7.6`)
- Replace `pip install -e ".[test]"` with `uv sync --extra test --frozen`
- Replace `pytest` with `uv run pytest`
- Replace `pip-audit` with `uv run pip-audit`
- `--frozen` enforces lockfile integrity in CI

### CI: `.github/workflows/pre-commit.yml`

- Replace `actions/setup-python` + pip with `astral-sh/setup-uv@v5`
- `uv sync --extra dev --frozen` then `uv run pre-commit run --all-files`

### `scripts/Dockerfile.dev`

```dockerfile
COPY --from=ghcr.io/astral-sh/uv:0.7.6 /uv /uvx /bin/
# ...
COPY pyproject.toml uv.lock ./
COPY src/ ./src/
# ...
RUN uv sync --extra dev --extra test --frozen
```

- Multi-stage copy of uv binary (no pip needed in container)
- `--frozen` ensures lockfile-verified installs
- Lockfile + pyproject.toml copied first for Docker layer caching

### Pre-commit Hook

New hook triggered when `pyproject.toml` changes:

```yaml
- id: uv-lock-check
  name: Check uv.lock is up-to-date
  entry: uv lock --check
  language: system
  pass_filenames: false
  files: ^pyproject\.toml$
```

Requires uv installed locally for committers who change dependencies.

### AGENTS.md

Update development setup commands to show uv workflow alongside pip.

## Supply Chain Protection Summary

| Attack                                           | Before                         | After                                         |
| ------------------------------------------------ | ------------------------------ | --------------------------------------------- |
| Version hijack (same version, different content) | Vulnerable                     | Blocked (hash mismatch)                       |
| Dependency confusion (internal name on PyPI)     | Vulnerable                     | Blocked (single index + lockfile pins source) |
| Transitive dep compromise                        | Vulnerable (not pinned)        | Blocked (every transitive dep hashed)         |
| Compromised CI install                           | No verification                | `--frozen` + hashes = reproducible            |
| Typosquatting                                    | Partially protected by == pins | Same + lockfile reviewable in PRs             |

## Backward Compatibility

- `pip install -e .` works (pip downloads `uv_build` automatically)
- `pip install -e ".[dev,test]"` works (optional-dependencies preserved)
- `pip install inference-endpoint` works (published package unchanged)
- Non-uv contributors: no workflow change except they don't get hash verification locally
